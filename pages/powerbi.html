Power BI

Classic Models in Power BI

Background

Purpose: In this page we are going to create several dashboards for "Classic Models"
that will help to set KPI's to track performance within our teams.


Technology

We will be taking advantage of Power BI Desktop, this tool gives you a bunch of visuals
to represent your data.





Dataset

Same data used in previous pages, Classic Models dataset loaded into MySQL, will be used.



Preliminary

Before starting the scenarios it is worth to look at the model diagram:

Now we can kick off the case scenarios.


Scenarios

Scenario 1:

The purchasing department who is reponsible for ensuring that our company mantain a correct level of
inventory, is asking if it is possible to have a breakdown of the following items:

  • Funnel of quantity in stock by product line.
  • Vertical column chart by product name showing current stock quantity.
  • Stock rotation card, this is calculated from quantity sold.

Finance team is also interested in having visibility over inventory revenue,and they are asking if it is
possible to achieve the following requirements:

  • Horizontal column chart reflecting inventory value by product name.
  • Inventory value card, this is calculated multiplying stock quantity by buy price.
  • Market value card, this is calculated multiplying stock quantity by MSRP.

Both teams have agreed on having slicers by product line, code and name. Please see below screenshot of
visual Classic Models Inventory:





Now we are going to interact with the dashboard, for that we set on slicer product line "Planes"
see how nicely features play out:





Another example to wrap up scenario 1, would be to select product name "1969 Ford Falcon":





Scenario 2:

Finance team reached out to us for creating a dashboard that fulfills the following requirements:

  • One slicer for the following categories, product line, product name and customer name.
  • A table sorted by order number containing customer name, product details and amount by order line.
  • A pie chart having total revenue by product line.
  • Horizontal column chart showing revenue by product.
  • Total revenue value card, total revenue since the company is operating.

Note that in the table there is a column "Concat" result of concatenating order number plus order line
I had to came up with that solution to be able to sort by order number. Please see the result of this visual:





Sales Team set slicer order number for order "10129", they are quite happy with
the level of detail this dashboard gives them:





Product line "Classic Cars" has the highest revenue, we want to break it down to see which product
and order line has the highest value:





Scenario 3:

Our colleagues from sales suggested a dashboard with the following requirements:

  • One slicer for product line and another one for customer name.
  • A pie chart showing yearly number of orders.
  • A table containing products, quantity ordered and sorted by total revenue.
  • A table listing customers, country, sales rep and sorted by revenue.
  • Map locating each customer city by revenue.

After considering above requirements we have created the following visual:

Product line "Trains" has the lowest revenue, and now we want to focus on it,
maybe this visual helps our sales to increase sales on "Trains":





Scenario 4:

For the last scenario sales team wants to have a dashboards with the following points:

  • One slicer for each of the following items, region, rep name and product line.
  • A matrix ordered by revenue showing regions and markets assigned to each sales rep.
  • A table for customers and their countries sorted by revenue.
  • Funnel listing regions by revenue.
  • Horizontal column chart ranking sales rep by revenue generated..
